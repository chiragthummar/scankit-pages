3. Keywords & ASO

App Store Connect's Keywords field is 100 characters total, comma-separated, no spaces between items. Don't repeat words that are already in the App Name or Subtitle — those are automatically indexed.

Primary keywords (paste directly)

qr,barcode,reader,code,generator,wifi,vcard,scanner,ean,upc,pdf417,qrcode,upi,epc,wallet

Words you do NOT need to add (already indexed)

  • "ai", "scanner", "generator" — in the App Name
  • "scan", "create", "share", "codes" — in the Subtitle

Localization suggestion

Add at least one second locale (e.g. English (UK) or Spanish (Mexico)) so you get a second 100-character keyword slot. The App Name/Subtitle/Keywords don't have to differ between locales.

6. Privacy nutrition labels

Apple's App Privacy section asks what data you collect. For this app the honest answer is nothing — every scan stays on-device, there's no account, and the only network call is an optional Safe Browsing check that sends just the URL being checked.

SectionAnswer
Do you or your third-party partners collect data from this app?No
If App Store Connect asks about Safe Browsing as third-party data collection — Google Safe Browsing uses hashed URL prefixes and is not a user-data collection service. You don't need to declare it under App Privacy. If you ever add analytics / crash reporting, come back and update this section.

10. TestFlight checklist

  • Upload a build via GitHub Actions (Deploy workflow → environment=testflight) or locally (bundle exec fastlane beta)
  • Wait 5–15 minutes for Apple to process it
  • Go to App Store Connect → TestFlight → the new build → answer Missing Compliance ("Standard encryption, exempt")
  • Add yourself + teammates to Internal Testing (no Apple review needed)
  • Add test info: beta app description + feedback email (akashmangukiya10@gmail.com)
  • Open the TestFlight app on your iPhone → tap Update to install the new build
  • Confirm version in Settings → About reads 1.2.0 (11) or newer
  • Smoke test: scan a QR, create a code, open the paywall, tap Restore Purchases

12. After submission

Review turnaround

Apple typically reviews a new iOS app within 24–48 hours. If you haven't heard back in 5 days, use Request Expedited Review.

If rejected

  • Read the rejection carefully in App Store Connect → Resolution Center
  • Most first-submission rejections hit these guidelines: 2.1 (App Completeness), 4.0 (Design), 3.1.1 (In-App Purchase), or 5.1.1 (Privacy)
  • Reply with clarifying info; you don't always need a new build
  • If a new build is required, increment the build number and upload again

After approval

  • Pin the 1.2.0 milestone tag in GitHub
  • Create a Phased Release for automatic updates (7-day ramp) if you used auto-release
  • Monitor crashes in Xcode Organizer → Crashes
  • Watch the Ratings & Reviews tab; reply to any 1-star reviews within 48h
